📌 Unit 1 — Introduction to Operating Systems

🖥️ What is an Operating System?

An OS is a software that acts as an interface between the user and the hardware.

  • Resource Manager — manages CPU, memory, and I/O
  • User Interface — provides a GUI or CLI
  • Program Execution — loads and runs programs
  • Error Detection — detects and handles errors

🗂️ Types of Operating Systems

TypeDescriptionExample
Batch OSJobs are processed in batches with no interactionEarly IBM

Real-Time OSStrict time deadlines, guaranteed responseRTOS, VxWorks
MultiprocessorMultiple CPUs in a single systemModern servers
MultiuserMultiple users at the same timeLinux servers
MultithreadedMultiple threads within a single processWindows, Linux
DistributedMultiple computers working as one unitCloud OS

🏗️ OS Structure

  • Monolithic Kernel: All services run in the kernel — fast but less secure. Example: Linux, UNIX
  • Microkernel: Only basic services run in the kernel, the rest run in user space — more secure but slower. Example: Mach, QNX
  • Layered Structure: The OS is divided into layers, and each layer uses services from the layer below.

A comparison of Monolithic and Microkernel structures definitely appears on exams!

⚙️ System Components

  • Process Management — Process creation, scheduling, termination
  • Memory Management — Memory allocate/deallocate karna
  • File Management — File create, delete, read, write
  • I/O Management — I/O devices control karna
  • Secondary Storage — Disk management
  • Protection & Security — Access control

🔄 Reentrant Kernels

A kernel is reentrant when multiple processes can execute kernel code simultaneously without interfering with each other.

  • Uses shared kernel code
  • Each process maintains its own kernel stack
  • Essential in multiprocessing environments

🔒 Critical Section Problem

When multiple processes access shared data, a race condition can occur.

Critical Section = the part of code where a shared resource is accessed.


3 Requirements for Solution:

  • Mutual Exclusion — only one process may be in the critical section at a time
  • Progress — if no process is in the critical section, a waiting process can enter
  • Bounded Waiting — no process waits forever

🔑 What is Mutual Exclusion?

Only one process should be in the critical section at a time. This is Mutual Exclusion.


Solutions:

  • Dekker's Algorithm
  • Peterson's Algorithm (common in exams!)
  • Semaphores
  • Monitors

🧪 Peterson's Solution (2 processes)

Two shared variables: flag[2] aur turn

Algorithm for Process Pi:

flag[i] = true; // "I want to enter the critical section"

turn = j; // "Let the other process go first"

while (flag[j] && turn == j); // Wait

--- CRITICAL SECTION ---

flag[i] = false; // "Main bahar hoon"

Peterson's solution satisfies all 3 conditions: Mutual Exclusion ✓, Progress ✓, Bounded Waiting ✓

🚦 Semaphores

A semaphore is an integer variable that supports two atomic operations:

  • wait(S) / P(S): S-- ; if S<0 then block
  • signal(S) / V(S): S++ ; if S≤0 then wake one blocked process

Types:

  • Binary Semaphore (Mutex): only 0 or 1 — used for mutual exclusion
  • Counting Semaphore: any integer — used for counting resources

Test and Set = a hardware solution for mutual exclusion. It is an atomic operation.

🍝 Dining Philosopher Problem

Five philosophers sit at a table. There are 5 chopsticks (forks). Each needs 2 chopsticks to eat.

Problem: If everyone picks up the left chopstick at the same time → DEADLOCK!


Solutions:

  • Have one philosopher pick the right chopstick first (asymmetric)
  • Allow at most 4 philosophers at the table
  • Pick up both chopsticks together (atomically)
  • Use semaphores

💈 Sleeping Barber Problem

In a barber shop: 1 barber, N waiting chairs.

  • If there is no customer → the barber sleeps
  • If a customer arrives and the barber is sleeping → he wakes up
  • If the waiting chairs are full → the customer leaves

Solution: Use semaphores for customers, barbers, and mutexes

🔄 Producer-Consumer Problem

Producer produces data, Consumer consumes it. There is a shared buffer.

  • mutex = 1 (binary semaphore for buffer access)
  • empty = N (counting semaphore for empty slots)
  • full = 0 (counting semaphore for filled slots)

Producer: wait(empty) → wait(mutex) → add item → signal(mutex) → signal(full)

Consumer: wait(full) → wait(mutex) → remove item → signal(mutex) → signal(empty)

📨 IPC — Inter Process Communication

Two models of communication between processes:

  • Shared Memory: a shared memory area where processes read/write directly. Fast but requires synchronization.
  • Message Passing: send() and receive() calls. Slower but safer. Examples: pipes, sockets, message queues.

⚡ Scheduling Algorithms

  • FCFS — First Come First Serve. Simple but can suffer from the Convoy Effect.
  • SJF — Shortest Job First. Minimum average waiting time. Non-preemptive.
  • SRTF — Shortest Remaining Time First. Preemptive SJF.
  • Round Robin — uses a time quantum. Preemptive and fair!
  • Priority — higher priority runs first. Can lead to starvation.
  • Multilevel Queue — multiple queues with different priorities.

Convoy Effect = in FCFS, a long CPU-bound process holds the CPU and delays all other processes!

🔴 Deadlock — Definition & Conditions

Deadlock = one or more processes are in a state where each is waiting for another permanently.


4 Necessary Conditions (Coffman's Conditions):

  • Mutual Exclusion — a resource can be used by only one process at a time
  • Hold and Wait — a process holds a resource while requesting another
  • No Preemption — a resource cannot be forcibly taken away
  • Circular Wait — a circular chain like P1→P2→P3→P1

All 4 conditions must hold together for deadlock! Break any one → no deadlock.

🛡️ Deadlock Handling Methods

  • Prevention: break one of the 4 conditions permanently
  • Avoidance: Banker's Algorithm — check for a safe state first
  • Detection & Recovery: allow deadlock, detect it, and recover
  • Ostrich Algorithm: ignore it (used in many real systems!)

🏦 Banker's Algorithm

Available resources are checked to see whether allocation will remain in a safe state.


Key matrices:

  • Allocation = Abhi kise kitna mila
  • Max = Maximum kitna chahiye
  • Need = Max - Allocation
  • Available = how much remains free in the system
Need[i][j] = Max[i][j] - Allocation[i][j]

Safe Sequence: Find a process whose Need ≤ Available, run it, free its resources, repeat.

✅ SOLVED — PYQ: 6 Tape Drives, Each Process needs max 2

Formula: System deadlock-free if: Resources > n(max_need - 1)

6 > n(2-1) = n

Maximum n = 5 (always deadlock free)

Minimum n = 1

With n=5: worst case each holds 1 drive = 5 drives held, 1 free → progress guaranteed!

🔁 Round Robin Quantum Size

n processes, each switch takes 's' seconds. CPU overhead from switching minimized when:

Quantum 'q' should be >> context switch time 's'

Minimum quantum such that each process gets turn every 't' seconds:

q ≥ (t - n×s) / n

📄 Paging

Divide physical memory into frames and logical memory into pages. Both are the same size!

Physical Address = Frame_Number × Frame_Size + Offset
Page Number = Logical Address / Page Size
Offset = Logical Address % Page Size

✅ SOLVED — PYQ: Logical=8 pages×1024 words, Physical=32 frames

Logical address bits = log₂(8×1024) = log₂(8192) = 13 bits

Page number bits = log₂(8) = 3 bits

Offset bits = log₂(1024) = 10 bits

Physical address bits = log₂(32) + 10 = 5+10 = 15 bits

🔍 TLB & Effective Access Time (EAT)

TLB = Translation Lookaside Buffer — Fast hardware cache for page table

EAT = α(TLB + Memory) + (1-α)(TLB + 2×Memory)

α = TLB hit ratio

✅ SOLVED — PYQ: Memory=200ns, TLB hit=75%

Without TLB: 2 × 200 = 400 ns (page table + data)

EAT = 0.75×200 + 0.25×400

= 150 + 100

EAT = 250 ns ✅

🗂️ Segmentation

Divide memory into variable-size segments (code, data, stack).

Physical Address = Base[segment] + Offset
Condition: Offset < Limit (else TRAP!)

🔄 Page Replacement Algorithms

  • FIFO: first in, first out replacement. Belady's Anomaly can occur!
  • Optimal (OPT): replace the page that will be used farthest in the future. Best but impractical.
  • LRU: Least Recently Used. Best practical algorithm.

Belady's Anomaly = in FIFO, increasing the number of frames can increase page faults! This does not happen in OPT or LRU.

🌐 Virtual Memory & Thrashing

  • Virtual Memory: use disk as an extension of RAM. The process is not fully loaded into memory.
  • Demand Paging: load a page only when it is needed
  • Page Fault: when the required page is not in memory
  • Thrashing: when a process generates so many page faults that little useful work is done. Often due to too many processes.
  • Working Set: the set of pages recently used — helps prevent thrashing

📦 Memory Allocation Strategies

  • First Fit: Pehla suitable hole. Fast.
  • Best Fit: choose the smallest suitable hole. Minimizes waste but is slower.
  • Worst Fit: choose the largest hole. Reduces fragmentation for large blocks.

Fragmentation:

  • Internal Fragmentation: Allocated memory > needed (extra waste inside)
  • External Fragmentation: free memory exists but is not contiguous

Paging solves External Fragmentation! Segmentation solves Internal Fragmentation!

Section A — Q1(b): Convoy Effect

Answer:

The Convoy Effect occurs in FCFS scheduling when one large CPU-bound process holds the CPU and smaller I/O-bound processes queue up behind it like a convoy. Result: low CPU utilization and poor response time.

Section A — Q1(c): Resource Partitioning vs Pool-based

Resource Partitioning Approach:

  • Divide resources into groups, each for a specific process class
  • No sharing between classes
  • Simple, predictable but wasteful

Pool-based Approach:

  • All resources in a common pool
  • Any process can request any resource
  • Better utilization but needs synchronization

Section C — Q5: Internal vs External Fragmentation

Internal Fragmentation:

In fixed-size partitions, allocated memory is larger than the actual need. Wasted space occurs inside the partition. This is internal fragmentation in paging.

External Fragmentation:

Free memory exists but is scattered (non-contiguous). The total free space may be enough but not contiguous. This is external fragmentation in segmentation. Solution = compaction.

Section C — Q7(b): Virtual Address Translation

Virtual space=2³², Physical=2¹⁸, Page size=4096=2¹², VA=11123456(hex)

Page offset bits = 12 (since page size = 2¹²)

Page number bits = 32-12 = 20 bits

VA = 11123456(hex) in binary (32 bits):

0001 0001 0001 0010 0011 | 0100 0101 0110

Page Number = 0x11123 | Offset = 0x456

System looks up page 0x11123 in page table → gets frame number → adds offset 0x456 → Physical Address

Section C — Q9(b): Dining Philosopher Problem

Problem:

5 philosophers, 5 chopsticks. Philosopher alternates between thinking and eating. Needs 2 chopsticks (left + right) to eat.

Deadlock Solution (Semaphore approach):

chopstick[5] = {1,1,1,1,1} as semaphores

Philosopher i: wait(chopstick[i]) → wait(chopstick[(i+1)%5]) → EAT → signal(chopstick[i]) → signal(chopstick[(i+1)%5])

Fix: One philosopher picks Right first, rest pick Left first. This breaks circular wait!
